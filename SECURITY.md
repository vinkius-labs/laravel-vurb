# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Laravel Vurb seriously. If you believe you have found a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

**Please do not open GitHub issues for security vulnerabilities.**

Instead, please send an email to [security@vinkius.com](mailto:security@vinkius.com). We will acknowledge your email within 48 hours, and we will send a more detailed response within 96 hours indicating the next steps in handling your report.

Please include the following information:

- Type of issue (e.g., SQL injection, XSS, authentication bypass)
- Full paths of source file(s) related to the issue
- Step-by-step instructions to reproduce the issue
- Any special configuration required to reproduce
- Impact of the issue and how an attacker might exploit it

We will provide a timeline for fixing the vulnerability and releasing a patch.
